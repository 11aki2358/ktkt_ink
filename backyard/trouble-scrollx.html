横に長いpreで

このpreのCSSは
.test {
  position: relative !important;
  z-index: -1;
  overflow: auto !important;
}

Situation

最初、headerタグ全体にはz-indexの指定をしていなかった(レコードとかジャケットにはz-index指定していたけれど)。

だけれど、prismを使ってコードを表示すると、コードの部分がヘッダーロゴよりも上に来てしまう。

ここで、


.language-css, 
.language-html{
  position: relative;
  z-index: -1;
}
という指定をした (z-indexの指定には、positionの設定も変える必要がある)。

コードがヘッダーロゴに重ならなくなった。 その一方、横に長いコードを表示するときに、スクロールバーが動かなくなってしまった(表示はされるが、バーが動かない)。

Reason

原因を調べた結果...

同じクラスでpositionとoverflowの設定をするのがマズい

という記述を見つけた。

(親子関係の要素で、親だけoverflow、子にだけpositionの設定をする、というのはOK。親だけposition, 子だけoverflowでもOK)

今回、prismではデフォルトで.language-なんちゃらにoverflowの設定がしてあった。 この設定だと、自分で書いたposition : relativeと競合してしまい、スクロールバーが動かなくなった。

ちょっと待って

でも、


test test test test test test test test test test test test test test test test test test test test test test test test test test test test test test 
ここのCSSは
.test2 {
  position: relative !important;
  z-index: 2;
  overflow: auto !important;
}
で、positionとoverflowどっちも設定しているけれど、ちゃんとスクロールは動く

スクロールバーが動かない原因はz-indexの値だった。
本文(z-index: 0)よりも下の層にスクロールバーが表示されたから、バーを操作できなかった。

positionとoverflowを同時に設定するのもマズいのだろうけれど、今回はもっと初歩的なミスでつまづいていたようだ。

Solution

よく考えなおしてみたら、コードやpreのz-indexをいじくってCSSをややこしくするよりも、headerのz-indexを手前に持ってくる方が楽じゃんね。

自分で書いたコード


.language-css, 
.language-html{
  position: relative;
  z-index: -1;
}
はばっさりカット。

コードやpreのz-indexをマイナスにする代わりに、ヘッダーのz-indexをプラス(z-index: 10;)にする。

これで、prismで飾られたコードがヘッダーよりも下に来る & 横に長いコードで横スクロールが効くようになった。
