Compare Static Hosting Services

静的サイトをホスティングできる無料サービスの比較。 ここでは、Netlify, Vercel, Firebaseを紹介。
個人サイトのサーバと言えば、国内のサービスが候補にあがると思う。 でも、「サーバ」(のようなものだけれど)静的ホスティングサービスというものもあって、こちらの方が便利な感じがあったので紹介する。

良い所

  • 最低限必要な機能は、無期限・無料で使える
  • 広告が挟まれない
  • 独自ドメインを持っていなくても、ドメイン発行してくれる
    独自ドメインを使いたい場合にもOK

難点

  • 海外のサービスなので、公式からは日本語での案内がない・少ない。
    (シンプルな英語なので、グーグル翻訳を駆使すればなんとかなる...) (多くの日本人が「○○のやり方」的な記事を書いているので参考に)

Overall

  • 全体的にお手軽。メールフォームを楽に付けるならばNetlify。認証・閲覧制限が必要な内容には不向き。
  • ちょうどよいバランスのVercel
  • 上級者向き・サイト公開だけではもったいないFirebase

Netlify

よいところ

  • 静的サイトだけれど、メールフォームは楽々付けられる。
  • gitHubと連携してデプロイできる。初期設定が簡単。
  • でも、git使わなくても、ブラウザにドラッグ&ドロップでもデプロイできる。
    「絶対にgit使いたくない! ターミナル見たくない! CLIって何?」という人に最適。

難点

  • 扱えるのは静的サイトのみ(html, CSS, javaScript)。
    サーバ側の設定は限定的。プログラムの使用は不可。
  • Basic認証など使うには、有料会員になる必要がある。
    (検索除けも閲覧制限も、html, CSS, javaScriptだけでは不完全かなぁ)。
  • ページの読み込みは遅めかも。

Vercel

よいところ :

  • gitHubと連携してデプロイできる。初期設定が簡単。
  • 無料でデータベースを使える
  • 無料でBasic認証を使える
  • phpやRubyなどのプログラムを使える : ...頑張ればフォーム認証を作れる
  • ページの読み込みが速い

Firebase

静的サイト(+ちょびっとphpとか)のためだけにfirebaseを使うのは大げさな印象。

よいところ

  • 無料で登録可能、Basic認証も無料
  • 静的サイト以外にも、データベースなどを扱うことが出来る
  • ページの読み込みが速いらしい

難点

  • 機能が多すぎて、初心者が使うには難しい
  • データベースを使うには、支払方法の登録が必要(有料版を使わなくても)。 使えるデータベースは、MySQLとは違う(日本語の資料が少ない気がする)。
  • npmやFirebase CLIのインストールが必要 : link
  • gitとの連携が難しそう(まだやっていない) : link link
    普通にFirebase CLIからページを更新(デプロイ)すればいいだけの話だけれど、gitにも履歴を保存しておきたいし、でもFirebase CLIとgitへのプッシュで二度手間になるのはめんどいしな～
