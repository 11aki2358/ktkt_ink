works.jsonをいじる

link
頑張ってベタ打ちする。いつか入力フォームを作りたいね
  • フォーマット
    
                {
                  "genre": [
                    "rock", "films"
                  ],
                  "year": 2026,
                  "rough": false,
                  "warn": [
                    "Spoiler",
                    "黒字はWARNING"
                  ],
                  "info": {
                    "comment": "適当なコメントをここに書く",
                    "editDate": 20260206,
                    "id": "a0000",
                    "modal": true,
                    "postDate": 20260205,
                    "title": "絵のタイトル"
                  }
                }
  • genre: 文字列
    複数選択可。検索キー。
    今のところ

    • "rock" : 音楽系
    • "2ji" : 二次元。漫画、アニメ、ゲーム、その他もろもろ
    • "films" : 映画系
    • "zatta" : その他、雑多

    のみ選択可
  • year: 数字
    制作年。
    検索キーとしても使える(が、今のところyear検索は無効としている)。コメントの下の方に表示される。
  • rough: bool
    雑な絵を描きました、という言い訳のために用意したプロパティ(今のところ使っていない)。
  • warn: 文字列
    警告
    ネタバレとか血とか薬とかの絵を描いた時用。記入した場合のみ、コメント欄に記載される。
  • info
    • comment: 文字列
      何かしらのコメント。
      長文でもOK。htmlタグを埋め込んでもOK。
    • editDate: 数字
      更新日時。
      年月日、十進数8桁で記入
    • id: 文字列
      コンテンツのID。
      各コンテンツに一意な文字列であること!!
      英数字とシンプルな記号のみで設定すること!!
      何かしら規則性ある文字列だといいなぁ(考え中)
    • modal: bool
      コンテンツの表示の仕方。
      TRUE: モーダルウィンドウで表示する。一枚絵。
      FALSE: コンテンツ固有のページに飛ぶ。複数枚の絵から成る場合、ワンクッション必要な内容の場合など。
    • postDate: 数字
      投稿日時。
      年月日、十進数8桁で記入
    • title: 文字列
      タイトル。
      長くてもいいけど、長すぎるとレイアウトが汚くなる。

画像やコンテンツ用のページを用意する

  • "modal": true
    • サムネ画像
      ファイル名 : works/images/work-id-thumb.png
      画像サイズ : 400px × 400pxくらい(正方形)
    • モーダル表示用の画像
      ファイル名 : works/images/work-id.png
      画像サイズ : 50 MiBよりも小さければ問題ない。クソデカ画像の場合でも、長辺8000px以内に抑えておくのが無難かな
  • "modal": false
    • サムネ画像
      ファイル名 : works/images/work-id-thumb.png
      画像サイズ : 400px × 400pxくらい(正方形)
    • コンテンツの個ページ
      ファイル名: works/work-id.html
      テンプレート : 後で作る
